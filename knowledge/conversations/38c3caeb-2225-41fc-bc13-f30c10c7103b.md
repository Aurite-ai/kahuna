# Built Customer Support AI Assistant

## Summary

The user requested help building an AI assistant that could answer questions about company policies using documentation files. The copilot explored the existing LangGraph project scaffold and found it was set up for arithmetic operations rather than customer support. 

The copilot then built a complete customer support system for "TechFlow Pro" (a B2B project management tool). This involved creating a comprehensive knowledge base with four documentation files covering FAQs, pricing plans, product features, and support policies. The copilot also developed a documentation reading tool that allows the AI agent to look up information from these files, and updated the system prompt to provide customer support guidelines.

The solution demonstrates a robust approach to building knowledge-based AI assistants, with clear separation between the knowledge base (markdown files) and the retrieval mechanism (the read_documentation tool). The agent was designed to always consult documentation before answering questions to ensure accuracy, rather than relying on the LLM's training data which could be outdated or incorrect.

## Decisions Made

1. **Create sample documentation content rather than wait for user content** - User chose to have sample content created to get the system working immediately
2. **Use markdown files for knowledge base** - Simple, readable format that's easy to maintain and update
3. **Build single read_documentation tool rather than separate tools per file type** - More flexible approach that allows the LLM to choose which document to read based on the question type
4. **Update system prompt to require documentation lookup before answering** - Ensures accuracy by preventing the agent from guessing or using potentially outdated training data
